示例文件 不良品报告

文件名：3QC前保险杠异响不良报告.pptx

文件类型：PowerPoint 演示文稿

内容：汽车前保险杠装配异响问题的不良品分析报告，包含发生状况、原因调查、改善对策等内容

操作方式：将 PPT 文件放入 input/ 目录，执行 python pipeline.py 即可自动完成全流程处理
1

MinerU 文档解析

将 PPT 解析为原始 Markdown + 提取图片

📥 输入

input/ └── 3QC前保险杠异响不良报告.pptx ← 原始PPT文件

⚙️ 处理过程

  1. 申请批量文件上传链接（MinerU API）
  2. 上传 PPT 文件到 MinerU 服务器
  3. 轮询等待解析完成（PPT → Markdown + 图片提取）
  4. 下载并解压解析结果

📤 输出

mineru_output/ └── 3QC前保险杠异响不良报告/ ├── 3QC前保险杠异响不良报告.md ← 原始解析的Markdown └── images/ ├── image_0.jpg ← PPT第1页图片 ├── image_1.jpg ← 不良品照片 └── image_2.jpg ← 原因分析图

📝 原始 Markdown 内容示例

# 此时内容结构松散，格式不统一 3QC前保险杠异响不良 ■发生日期：2024年3月15日 ■发生场所：总装车间 ■不良现象：前保险杠装配后行驶中产生异响 ■车型：3QC ■零件名称：前保险杠总成 照片： ![](images/image_1.jpg) 原因分析： 焊接点位置偏差导致装配间隙过大 对策： 调整焊接夹具定位销
注意：此时的 Markdown 只是 PPT 内容的"平铺"，没有按照标准模板结构化，无法直接用于知识库
2

Kimi 文档整理

按照五部分模板重新组织内容结构

📋 模板结构

Kimi 使用预定义的不良报告-硬件类模板，包含五个固定部分：

第一部分：发生状况 (5W2H) 第二部分：事实把握 第三部分：原因调查 第四部分：改善对策 第五部分：水平展开

⚙️ 处理过程

  1. 读取 Stage 1 产出的原始 Markdown
  2. 调用 Kimi API，传入模板结构作为 System Prompt
  3. Kimi 分析内容，将信息归类到对应的模板章节
  4. 从文件名中提取车型代号（如 3QC）作为标题

📤 输出

kimi_output/ └── batch_20240513_143022/ └── processed_3QC前保险杠异响不良报告.md ← 结构化Markdown

📝 整理后的 Markdown 内容示例

Pipeline 阶段总结

各阶段的作用与输出

阶段 作用 输入 输出
Stage 1 MinerU 将PPT/Word/PDF解析为原始Markdown input/*.pptx mineru_output/{文件名}/
Stage 2 Kimi 按五部分模板重组内容结构 mineru_output/*.md kimi_output/batch_{时间戳}/
Stage 3 图片处理 移动图片、更新链接、处理标题 kimi_output/*.md + 图片 output/*.md（最终产物）
Stage 4 归档 保存中间产物、清空目录 所有中间目录 data/{时间戳}/
使用方式：只需将待处理文件放入 input/ 目录，运行 python pipeline.py，最终的结构化 Markdown 就会自动生成到 output/ 目录，可直接用于知识库录入或前端展示。
